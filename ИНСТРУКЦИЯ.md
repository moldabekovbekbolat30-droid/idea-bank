# 📘 Инструкция — Банк идей

Простая веб-форма: сотрудники присылают идеи/проблемы без регистрации, владелец смотрит их в приватном обзоре.

## 1. Настроить хранилище (Firebase, один раз)

1. Зайти на https://console.firebase.google.com → «Add project» → создать проект (например `idea-bank`).
2. В проекте: **Build → Firestore Database → Create database** → выбрать регион → режим **production**.
3. Во вкладке **Rules** вставить:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /ideas/{doc} {
         allow read, create: if true;
         allow update, delete: if false;
       }
     }
   }
   ```
   (запись открыта всем — это форма без авторизации, как и описано в PRD; удаление/изменение запрещено даже случайно)
4. **Project settings → General → Your apps → Web (`</>`)** → зарегистрировать приложение → скопировать объект `firebaseConfig`.
5. Открыть `index.html`, найти `FIREBASE_CONFIG` и вставить свои значения вместо `"ВСТАВЬ_СЮДА"`.

## 2. Задать свой код доступа к обзору

В `index.html` найти строку:
```js
const ACCESS_CODE = "идея2026";
```
Заменить на свой код. Это просто замок в интерфейсе (см. ограничения в `prd.md`), не полноценная защита — не хранить в системе конфиденциальные данные компании.

## 3. Открыть локально

```bash
python3 -m http.server 5500
```
→ http://localhost:5500

## 4. Опубликовать на GitHub Pages

```bash
git init
git add .
git commit -m "Банк идей: форма сбора идей сотрудников"
git branch -M main
git remote add origin <ссылка-на-твой-репозиторий>
git push -u origin main
```
Затем в репозитории: **Settings → Pages → Source: Deploy from a branch → main / (root)**.

## 5. Как пользоваться

- **Сотрудники**: открывают ссылку, пишут идею, по желанию — имя/отдел, жмут «Отправить идею».
- **Владелец**: внизу страницы — кнопка «Обзор (для владельца)», вводит код доступа, видит список идей от новых к старым и счётчик.
