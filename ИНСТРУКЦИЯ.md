# 📘 Инструкция — Банк идей

Простая веб-форма: сотрудники присылают идеи/проблемы без регистрации, ответы попадают в Google Форму, привязанную к аккаунту владельца.

## Сайт

🌐 https://moldabekovbekbolat30-droid.github.io/idea-bank/

## Как пользоваться

- **Сотрудники**: открывают ссылку, пишут идею, по желанию — имя/отдел, жмут «Отправить идею».
- **Владелец**: открывает свою Google Форму «Банк идей» → вкладку **«Ответы»** — там список всех идей, счётчик и сводка. Доступ виден только с аккаунта, на который зарегистрирована форма (moldabekovbekbolat30@gmail.com) — вход защищён самим Google-аккаунтом, отдельный код не нужен.

## Как это устроено технически

- Сайт (`index.html`) — статическая страница на GitHub Pages, без своего бэкенда.
- При отправке формы браузер посылает данные напрямую в Google Форму (`formResponse` эндпоинт) через `fetch`.
- Все технические ID (`GOOGLE_FORM_ACTION`, `ENTRY_IDEA`, `ENTRY_NAME`) зашиты в `index.html` — они не секретны, это стандартный публичный способ отправки в Google Forms.

## Изменить форму (добавить вопрос, поменять текст)

Если понадобится поменять вопросы в Google Форме — учти, что технические ID полей (`entry.XXXXX`) при этом могут не измениться, но проверить стоит:
1. Открой форму → ⋮ → «Форма с предварительным заполнением».
2. Впиши в поля любые тестовые значения → «Получить ссылку».
3. В скопированной ссылке будут видны `entry.XXXXX=значение` — это и есть ID полей.
4. Обнови `ENTRY_IDEA` / `ENTRY_NAME` в `index.html`, если ID изменились.

## Открыть локально для разработки

```bash
python3 -m http.server 5500
```
→ http://localhost:5500

## Опубликовать изменения на GitHub Pages

```bash
git add -A
git commit -m "описание изменений"
git push
```
Сайт обновится по тому же адресу через 1–2 минуты.
